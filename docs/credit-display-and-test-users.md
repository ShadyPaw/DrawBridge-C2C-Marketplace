# 信用分展示与测试用户变更说明

## 本次目标

1. 统一商品购买页与用户主页的信用分展示样式。
2. 删除用户主页上的“用户等级”标签，仅保留“信用分”和“私聊”主操作。
3. 增加一组中风险、高风险演示用户，便于验证风控与页面展示。

## 前端改动

### 1. 商品详情页 `ProductDetail.vue`

- 将卖家信息区的信用分改为统一的徽章样式。
- 根据 `sellerCreditScore` 做三段式颜色区分：
  - `>= 100`：绿色
  - `60 - 99`：黄色
  - `< 60`：红色
- 保留原有购买前风控校验逻辑。
- 保留 `403` 时的红色高危拦截模态框。
- 清理页面中的历史乱码中文。

### 2. 用户主页 `UserProfile.vue`

- 删除“等级”展示。
- 头部仅保留：
  - 信用分徽章
  - 私聊按钮
- 举报入口下移为次级操作，整体样式与商品详情页统一。
- 清理页面中的历史乱码中文。

### 3. 个人中心 `MyCenter.vue`

- 同步把个人中心中的信用分展示改成同一套颜色规则，避免项目内部视觉口径不一致。
- 清理页面中的历史乱码中文。

## 演示数据改动

修改文件：`backend/src/main/java/com/cargoco/service/impl/DataInitServiceImpl.java`

### 新增 5 个中高风险测试用户

- `risk_demo_1` / 被举报较多的老周 / 信用分 `38`
- `risk_demo_2` / 新号急售的小许 / 信用分 `49`
- `risk_demo_3` / 交易记录稀少的小吴 / 信用分 `63`
- `risk_demo_4` / 中风险样本阿杰 / 信用分 `76`
- `risk_demo_5` / 观察中的小林 / 信用分 `88`

### 同时优化了初始化逻辑

- 原先所有演示用户默认都是 `100` 分，不利于测试。
- 现在普通演示用户也做了信用分梯度分配。
- 当初始化接口重复执行时：
  - 已存在的演示用户会被刷新昵称、头像、邮箱、手机号和信用分
  - 不会再因为“用户已存在”而保留旧的 `100` 分状态
- 为了方便页面验证，前几条演示商品会优先挂到中高风险测试用户名下。

## 当前展示规则

### 信用分颜色规则

- 绿色：信用分 `>= 100`
- 黄色：信用分 `60 - 99`
- 红色：信用分 `< 60`

这套规则目前已经统一到：

- 商品详情页卖家信息
- 用户主页头部
- 个人中心信用分卡片

## 建议的验证方式

1. 执行一次演示数据初始化接口。
2. 刷新首页或商品列表，进入不同卖家的商品详情页。
3. 检查卖家信用分标签颜色是否随分数变化。
4. 打开对应用户主页，检查信用分颜色是否与商品页一致。
5. 检查用户主页顶部是否只剩“信用分”“私聊”两个核心元素。

## 额外说明

这次没有去修改公开信用分的业务口径，只是统一了展示层和演示数据层，避免再次把 AI 风控分和公开信用分混用。
